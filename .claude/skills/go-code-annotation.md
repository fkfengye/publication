# Go Code Annotation Skill

## 触发条件

用户要求对 Go 代码添加注释、解释代码时，自动使用本技能。

## 注释规范

### 1. 文件顶部语法速查块

每个 `.go` 文件顶部添加一个 `/* Go 语法速查 */` 块，集中列出该文件用到的所有语法点：

```go
/*
Go 语法速查:

interface 接口类型: 定义方法签名，实现是隐式的（无需 implements 关键字）
func 方法签名: func Name(params) returns，多返回值用逗号分隔
*Type 指针类型: &取地址，*解引用，指针避免大对象拷贝
context.Context 请求上下文，Go 惯例放在第一个参数
[]Type 切片类型
包名.类型名 访问其他包的导出类型
*/
```

### 2. 代码块功能描述

每个代码块（type/struct/func/const 等）单独一行 `//` 备注其功能：

```go
// 定义 LLMProvider 接口，约束大模型交互必须实现的方法
type LLMProvider interface {
    ...
}

// 构造函数，初始化 AgentEngine
func NewAgentEngine(...) *AgentEngine {
    ...
}
```

### 3. 多行注释统一用 /* */

当需要多行描述时，统一使用 `/* ... */` 块注释风格，不使用逐行 `//`：

```go
/*
	这是多行注释的第一行
	这是第二行
	...
*/
```

### 4. 单行注释保持简洁

单行注释只写功能描述，不写语法（如 `// 定义结构体` 而非 `// type struct 定义结构体`）：

```go
// 初始化对话历史，包含系统提示和用户输入
contextHistory := []schema.Message{...}

// 如果没有工具调用，表示任务完成，退出循环
if len(responseMsg.ToolCalls) == 0 {
    ...
}
```

## 注释层级

| 层级 | 内容 | 格式 |
|------|------|------|
| 文件顶部 | 该文件用到的 Go 语法速查 | `/* ... */` 多行块 |
| 代码块 | 功能描述 | `// 单行` |
| 关键逻辑 | 业务逻辑说明 | `// 单行` |

## 注意事项

- 代码本身保持干净，不逐行备注语法
- 注释只描述"做什么"，不描述"怎么做"（代码本身已清晰表达）
- 单行注释和代码之间留一个空格
- 保持注释简洁，避免冗长描述
